## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engine → walker/elephant lifecycle | Killing and relaunching long-running daemons; a bad restart can leave a blank/stale UI |
| engine → thunar daemon lifecycle | Restarting the background daemon without disturbing visible windows |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Denial of Service | bounded-poll restart loop (walker/thunar) | low | mitigate | Cap the poll iteration count and fall through to relaunch (or SIGKILL) after the cap, so a process that never exits cannot hang the switch. |
| T-03-02 | Tampering | walker relaunch without elephant health gate | low | mitigate | Verify elephant socket + `elephant version` + walker/elephant version compatibility before declaring the restart complete, so a stale/mismatched elephant is not mistaken for a themed walker (Pitfall W2). |
| T-03-SC | Tampering | package installs | low | accept | No package installs in this plan (handled in Plan 01-01). Below the `high` block threshold. |
</threat_model>
