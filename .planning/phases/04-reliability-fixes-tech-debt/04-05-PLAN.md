<objective>
Close the single Phase 4 verification gap: fish adopts node tooling but never activates it on a fresh shell (04-REVIEW.md Critical CR-01). fish sources `conf.d/nvm.fish` (its auto-activation guard) BEFORE `config.fish` sets `nvm_data`/`nvm_default_version`, so the guard sees no default version and silently skips activation — leaving `node`/`npm`/`npx` off PATH in every clean fish shell despite D-10 promising "working node tooling" as day-one parity. This plan adds an explicit, guarded `nvm use` activation in config.fish (the 04-REVIEW.md CR-01 primary fix) and documents the one-time fresh-machine Node provisioning step (the gap's secondary `missing:` item).

Purpose: Restore D-10 fish node-tooling parity (FIX-03's fish adoption) so opening a fresh terminal gives a working node toolchain — the exact "de-risk the base" outcome Phase 4 targets.

Output: A fixed config.fish that activates node on a clean shell, an install.sh setup-doc line for fresh-machine Node provisioning, and 04-05-SUMMARY.md.

Lineage: This gap-closes work delivered by 04-04-PLAN.md (fish adoption). Those files already exist on disk and are committed; this plan modifies config.fish in place — it does NOT replan or revisit 04-04's shell decision, benchmark, or wiring, all of which verified correctly.
