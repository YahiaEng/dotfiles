<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disposable env → host machine | The NOPASSWD sudo needed for zero-prompt runs must never escape the throwaway container/VM onto the host or into the repo |
| remote repo → fresh environment | The fresh env clones and executes the repo's scripts as (passwordless) root-capable; a committed secret or a malicious untracked artifact would run here |
| harness packages → host | podman/qemu/libvirt are host installs whose integrity is delegated to pacman |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04-NOPASS | Elevation of Privilege | NOPASSWD:ALL sudoers left active/committed | high | mitigate | Drop-in is generated INSIDE the ephemeral container at runtime (Task 1) and documented as VM-only + discarded (Task 2); repo grep confirms no committed sudoers/NOPASSWD file — no path to a persistent-machine EoP |
| T-03-04-DOS | Denial of Service | partial container/VM run leaving an inconsistent env | low | mitigate | The env is disposable (fresh container per run, snapshot VM); Wave-1 guards make a re-run converge — a partial run is discarded, not compounded |
| T-03-SC | Tampering | podman/qemu/libvirt/edk2-ovmf/dnsmasq host installs | medium | accept | All official `extra` repo, verified via `pacman -Si` in RESEARCH; no npm/pip/cargo and no AUR for the harness; `iptables-nft` (nonexistent) explicitly avoided in favor of `iptables`; pacman keyring GPG chain enforces integrity |
</threat_model>
